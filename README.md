## Overview

Apeaksoft Phone Mirror is designed to project a phone screen onto a larger display while keeping everyday control in a desktop-friendly environment. It works with Android and iOS mirroring, which makes it suitable for presentations, demos, app testing, content checks, and routine device management on Windows, macOS, and Linux.

The app goes beyond simple viewing by adding capture and interaction tools that streamline hands-on work. Wireless casting, USB mirroring, local recording, and virtual camera output support both live use and media-oriented tasks. For more advanced setups, the WebSocket API lets scripts and external automation tools tie into the mirror session.

## How to Use It

A common session usually follows this flow:

1. Open Apeaksoft Phone Mirror on your desktop.
2. Connect an Android or iOS device through wireless casting or a USB cable.
3. Decide whether you only need viewing, or also want device control, recording, or virtual camera output.
4. Use the keyboard and mouse controls to operate the phone from your computer.
5. Save the mirrored session locally when you need MP4 or GIF output.
6. If you are building automation, connect to the WebSocket API from your script or integration layer.

For streaming workflows, select the virtual camera output and point your broadcasting app to it as the video source.

## Configuration

Settings are managed through the desktop application interface, mainly in the connection and preferences screens. Common options cover language, connection mode, recording behavior, and output handling.

Example settings layout:

```ini
[general]
language = auto
connection = wireless
control = enabled

[recording]
format = mp4
capture_audio = false

[automation]
websocket_api = enabled
```

If your build stores preferences locally, check the application profile folder for saved settings and session data.
